# chordcanvas 仕様書

## 目的
`chordcanvas` は、ローカルで動作するギター用コードダイアグラムエディタである。  
コードダイアグラムの生成、押弦編集、コード名判定、オンコード判定、複数コードの配置編集をブラウザ上で行えるようにする。

---

## 実装方針
- フロントエンドのみで構成する
- ローカルで動作する Web アプリとして実装する
- 技術構成は Vite + React + TypeScript
- バックエンド、データベース、Python は使わない
- 状態管理は当面 React 標準機能で行う
- コードダイアグラム表示は SVG を基本とする
- レイアウト編集は通常の HTML / CSS ベースで実装する

---

初期実装では以下を対象とする。

- 単一コードダイアグラムの表示
- プルダウンからのコード生成

- コード名の自動判定
- オンコード判定
- 基本 5 フレット表示、必要に応じて拡張
- 複数コードの横並び表示
- 複数の歌詞行に対するコード配置
- project 内で再利用するコードストック
- コードダイアグラム同士の間隔調整
- 歌詞行に対するコード位置の手動調整
- プロジェクト JSON の書き出し / 読み込み
- レイアウト PDF の書き出し

初期実装では以下は対象外とする。

- サーバー保存
- ユーザーアカウント
- 外部コード辞書の自動収集
- 画像書き出し
- 印刷最適化
- 複雑な自動譜割り
- 高度なスナップ編集
- 楽譜記法対応

- 左利き表示
- 別チューニング対応

## 画面構成
画面は大きく以下の領域で構成する。

### 1. コード生成パネル
- ルート音選択
- コード種別選択
- 候補フォーム選択
- 選択結果は次に追加する current chord に即時反映する
- 現在の押弦をコードストックへ追加できる

### 2. コードダイアグラム編集パネル
- 現在のコードダイアグラム表示
- 編集開始フレット指定
- 編集対象フレット数指定
- 各弦各フレットのクリック編集
- 開放弦表示
- ミュート表示
- 押弦マーカー内のコードトーン degree 表示
- コード生成で選択したフォームと current chord の押弦編集を同じ UI で行う
- レイアウト上の既存コードは、明示的に編集開始したときだけ current chord と接続する

### 3. コード情報表示パネル
- 現在のコード名
- 現在のコード名の手動上書き
- 候補コード名一覧
- ベース音
- 構成音
- ユニーク音
- 発音音

### 4. コードストックパネル
- project 内で再利用するコード一覧
- ストック済みコードのダイアグラム表示
- ストック済みコードの表示名保持
- ストック済みコードの削除
- 選択中のレイアウト行への追加
- 同一押弦のコードは重複登録しない

### 5. レイアウト編集パネル
- 複数の歌詞行
- 行追加
- 複数コードダイアグラムの横並び表示
- 各コードの表示名編集
- 選択中コードの配置行変更
- 各コードの位置調整
- 各コード間の間隔調整

### 6. プロジェクト操作
- 現在の project 状態を JSON として書き出す
- JSON ファイルから project を読み込む
- 読み込み時は現在の編集中 state を置き換える
- 現在のレイアウトを PDF として書き出す
- PDF 出力は A4 縦向きとし、画面の devicePixelRatio に依存せず印刷向けの解像度を確保する

- D
- D#
- E
- F
- F#
- G
- G#
- A
- A#
- B

- 6

### コード構成音定義
コード種別ごとにルートからの相対音程セットを内部辞書として持つ。

例:
- major = 0, 4, 7
- minor = 0, 3, 7
- 5 = 0, 7
- sus2 = 0, 2, 7
- sus4 = 0, 5, 7
- dim = 0, 3, 6
- aug = 0, 4, 8
- 6 = 0, 4, 7, 9
- m6 = 0, 3, 7, 9
- 7 = 0, 4, 7, 10
- maj7 = 0, 4, 7, 11
- m7 = 0, 3, 7, 10
- m7b5 = 0, 3, 6, 10
- dim7 = 0, 3, 6, 9
- add9 = 0, 4, 7, 14
- maj9 = 0, 4, 7, 11, 14
- m9 = 0, 3, 7, 10, 14
- 7sus4 = 0, 5, 7, 10

### フォーム候補
コード選択時、内部辞書から候補フォームを表示する。  
初期実装では代表的なフォーム辞書を持つ方式を採用する。

- ルート音ごとに移調可能なフォームを持つ
- 候補が複数ある場合はプルダウンまたは切替 UI で選択できる
- 第一候補を初期表示する

## コードダイアグラム表示仕様

### 基本構造
- ダイアグラムは左がヘッド側になる向きで表示する
- ダイアグラムは上が 1 弦、下が 6 弦になる向きで表示する
- 6 本の弦を横線で表示する
- 基本は 5 フレット分の縦線を表示し、必要に応じて増やす
- 押弦位置は丸で表示し、丸内にはコードトーン degree を表示する
- 開放弦は `O`
- ミュート弦は `X`
- 表示中の各フレット番号を表示する

### フレット表示範囲
ダイアグラムには、現在の押弦状態を含む範囲を基本 5 フレットで表示し、必要な場合のみ拡張する。

#### 算出方法
- `X` と `0` を除いた押弦フレット番号を収集する
- 最小フレット値を `minFret`
- 最大フレット値を `maxFret`
- 必要フレット数は `maxFret - minFret + 1`

#### 表示ルール
- 押弦が 1〜4 フレット内に収まる場合はナット基準表示とする
- 最小押弦フレットが 5 以上の場合も開始フレット番号の専用ラベルは表示しない
- ハイポジションでは表示中の各フレット番号で位置を示す
- 表示フレット数の最小は 5 とする
- 実使用範囲が 5 を超える場合は、その範囲に合わせて表示フレット数を増やす
- 押弦がない場合も 1〜5 フレットを表示する

### 表示例
- `x32010` → ローポジション表示
- `x68676` → 6〜10 フレット表示
- `x68899` → 6〜10 フレット表示

## 押弦編集機能

### 編集可能要素
各弦について、以下のいずれかを設定できる。

- ミュート
- 開放弦
- 任意のフレット押弦

### 編集ルール
- 1 弦につき 1 つだけ発音状態を持つ
- 別のフレットを選択したら、その弦の以前の状態は置き換える
- ミュートと開放弦も他の状態と排他的に扱う
- 押弦入力 UI では開始フレットと編集対象フレット数を指定できる
- 押弦入力 UI はコード生成後のフォーム調整と current chord 編集で共通に使う
- レイアウト上の既存コードは、明示的に編集開始した場合だけ押弦入力 UI の変更を反映する

### 編集結果の反映
押弦編集後、以下を再計算する。

- 実際に鳴る音
- ユニーク音
- ベース音
- コード名候補
- オンコード表記
- 表示フレット範囲

---

## 音名計算

### 基本
各弦の開放音とフレット番号から実音を算出する。

### 処理
- 開放弦の音高を基準に半音加算する
- 音名は 12 音クラスとして扱う
- コード判定ではオクターブ違いは同一音として扱う

### 出力
- 各弦の実音一覧
- ユニーク音一覧
- ベース音

---

## コード名判定

### 判定対象
現在鳴っている音からコード名候補を判定する。

### 基本手順
1. 鳴っている音を集める
2. ユニーク音に変換する
3. すべてのユニーク音をルート候補として試す
4. 各ルート候補について相対音程セットを計算する
5. 内部辞書のコード種別と照合する
6. 一致する候補を一覧化する
7. 優先順位に従って第一候補を決定する

### 優先順位
- 構成が単純なものを優先
- 一般的なコード名を優先
- ベース音との整合性を考慮する
- 不要に複雑な解釈は下位候補に回す

### 表示
- 第一候補を現在のコード名として表示する
- 第二候補以下は候補一覧に表示する

---

## オンコード判定

### 判定方法
- 実際に鳴っている最低音をベース音とする
- コード本体のルート音とベース音が異なる場合は `/bass` 表記にする

### 例
- 本体が C、最低音が E → `C/E`
- 本体が D、最低音が F# → `D/F#`

### 注意
- コード本体判定とベース音判定は分離して扱う
- 最低音を先にルートと決め打ちしない

---

## 候補コード名一覧
複数解釈が成立する場合は候補一覧を表示する。

### 例
- `C/E`
- `Em(#5)`
- `C`

初期実装では 3 件程度まで表示できればよい。

---

## 複数コード管理

### 概要
単一コードだけでなく、複数のコードダイアグラムを並べて管理できるようにする。

### 各コードブロックが持つ情報
- 一意な ID
- 押弦データ
- 表示コード名
- 所属行 ID
- 横位置オフセット
- 並び順
- 間隔調整値

ベース音、表示開始フレット、表示フレット数は押弦データから導出する。コード名は押弦から自動判定しつつ、必要に応じて表示用の上書き名を持てるようにする。

---

## レイアウト編集機能

### 基本
複数のコードダイアグラムを横方向に並べて表示する。

### できること
- コードブロックの追加
- コードブロックの削除
- コードブロックの複製
- コードブロックの並び替え
- レイアウト行の追加
- 選択中コードを明示的に編集対象へ切り替える
- 選択中コードの配置行変更
- コードブロックの左右位置調整
- コードダイアグラム同士の間隔調整

### 位置調整方法
初期実装では以下のいずれか、または両方を採用する。

- ドラッグによる移動
- 数値または簡易操作による左右オフセット調整

---

## コードストック機能

### 基本
project 内で再利用したいコード押弦をストックとして保持できるようにする。

### 仕様
- 現在選択中のコード押弦をストックへ追加できる
- 現在選択中のコード表示名もストックへ引き継げる
- ストックしたコードは任意のレイアウト行へ追加できる
- ストック項目は削除できる
- 同一押弦のコードは 1 回だけ保存する
- ストックは project JSON に含めて保存する

---

## 歌詞行との対応

### 基本
複数の歌詞行を表示し、各行の上にコードダイアグラムを配置できるようにする。

### 仕様
- 各行の歌詞は単純な 1 行テキスト入力とする
- 通常表示はレイアウト上の歌詞テキストとして見せ、対象行を操作したときだけその場で編集状態に入る
- 歌詞入力欄に含まれる空白は表示側でも保持し、手動の譜割り調整に使えるようにする
- PDF 出力時も画面上の歌詞表示位置と揃うように、表示基準の位置関係を維持する
- レイアウト行は追加できる
- コードブロックはいずれか 1 つの行に属する
- コードブロックは所属する歌詞行の上に配置する
- 追加先の行を選択できる
- 選択中コードの所属行を変更できる
- 各コードの左右位置を手動調整できる
- 譜割りの自動解析は行わない

### 目的
- 語頭や音節に合わせてコード位置を手で合わせられるようにする
- コード間隔と歌詞位置の両方を調整できるようにする

---

## 状態管理

### 最低限管理する状態
- 現在選択中のルート音
- 現在選択中のコード種別
- 現在選択中のフォーム候補
- 次に追加する current chord の押弦状態
- 次に追加する current chord の表示コード名
- 現在選択中のコードブロック ID
- current chord と接続中のコードブロック ID
- 追加先として選択中のレイアウト行 ID
- project 内のコードストック
- コードブロックごとの押弦状態
- 押弦入力 UI の開始フレット
- 押弦入力 UI の編集対象フレット数
- 現在のコード名候補一覧
- 複数コード配列
- 複数の歌詞行
- コードブロックと歌詞行の対応関係
- レイアウト位置情報

### 保存
サーバー保存は行わない。  
ローカルファイルとして project JSON の書き出し / 読み込みをサポートする。

### project JSON の扱い
- project 全体の状態を 1 つの JSON ドキュメントとして保存する
- ファイルには format と version を持たせる
- 読み込み時は format / version / 必須 state を検証する
- 不正な JSON や未対応 version は読み込みエラーとして扱う
- `currentChordName` が存在しない旧 project JSON は空文字列として扱う
- `stockChords` が存在しない旧 project JSON は空配列として扱う

## データ構造方針

### 押弦データ
各弦について以下のいずれかを持つ。

- `x`
- `0`
- 正の整数フレット番号

例:
- `["x", 3, 2, 0, 1, 0]`

### コードブロック
例:
- `id`
- `fretting`
- `displayName`
- `rowId`
- `xOffset`
- `spacing`

ベース音、表示開始フレット、表示フレット数は押弦データから導出する。コード名は押弦から自動判定しつつ、表示用の上書き名を保持できる。

### レイアウト行
例:
- `id`
- `lyrics`

### コードストック
例:
- `id`
- `fretting`
- `displayName`

### project ファイル
例:
- `format`
- `version`
- `exportedAt`
- `state`

### コードタイプ辞書
コード種別名と相対音程配列を対応させる。
