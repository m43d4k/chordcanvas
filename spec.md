# chordcanvas 仕様書

## 目的
`chordcanvas` は、ローカルで動作するギター用コードダイアグラムエディタである。  
コードダイアグラムの生成、押弦編集、コード名判定、オンコード判定、複数コードの配置編集をブラウザ上で行えるようにする。

---

## 実装方針
- フロントエンドのみで構成する
- ローカルで動作する Web アプリとして実装する
- 技術構成は Vite + React + TypeScript
- バックエンド、データベース、Python は使わない
- 状態管理は当面 React 標準機能で行う
- コードダイアグラム表示は SVG を基本とする
- レイアウト編集は通常の HTML / CSS ベースで実装する

---

## スコープ
初期実装では以下を対象とする。

- 単一コードダイアグラムの表示
- プルダウンからのコード生成

- オンコード判定
- 必要フレット数だけの表示
- 複数コードの横並び表示
- コードダイアグラム同士の間隔調整
- 歌詞行に対するコード位置の手動調整

初期実装では以下は対象外とする。

- サーバー保存
- ユーザーアカウント

- PDF 書き出し

- 印刷最適化

- 楽譜記法対応

## コード生成機能

### ルート音選択
ルート音はプルダウンから選択する。

対応する音名:
- C
- C#
- D
- D#
- E
- F
- F#
- G
- G#
- A
- A#
- B

表記は当面シャープ系で統一する。

- 6

### コード構成音定義
コード種別ごとにルートからの相対音程セットを内部辞書として持つ。

例:
- major = 0, 4, 7
- minor = 0, 3, 7
- 5 = 0, 7
- sus2 = 0, 2, 7
- sus4 = 0, 5, 7
- dim = 0, 3, 6
- aug = 0, 4, 8
- 6 = 0, 4, 7, 9
- m6 = 0, 3, 7, 9
- 7 = 0, 4, 7, 10
- maj7 = 0, 4, 7, 11
- m7 = 0, 3, 7, 10
- m7b5 = 0, 3, 6, 10
- dim7 = 0, 3, 6, 9
- add9 = 0, 4, 7, 14
- maj9 = 0, 4, 7, 11, 14
- m9 = 0, 3, 7, 10, 14
- 7sus4 = 0, 5, 7, 10

### フォーム候補
コード選択時、内部辞書から候補フォームを表示する。  
初期実装では代表的なフォーム辞書を持つ方式を採用する。

- ルート音ごとに移調可能なフォームを持つ
- 候補が複数ある場合はプルダウンまたは切替 UI で選択できる
- 第一候補を初期表示する

## コードダイアグラム表示仕様

### 基本構造
- 6 本の弦を縦線で表示する
- 必要なフレット数だけ横線を表示する
- 押弦位置は丸で表示する
- 開放弦は `O`
- ミュート弦は `X`

### フレット表示範囲
ダイアグラムには、現在の押弦状態に必要なフレット範囲だけを表示する。

#### 算出方法
- `X` と `0` を除いた押弦フレット番号を収集する
- 最小フレット値を `minFret`
- 最大フレット値を `maxFret`
- 必要フレット数は `maxFret - minFret + 1`

#### 表示ルール
- 押弦が 1〜4 フレット内に収まる場合はナット基準表示とする
- 最小押弦フレットが 5 以上の場合は開始フレット番号を表示する
- 表示フレット数は必要範囲を基本とする
- 必要に応じて編集余白として 1 フレット追加してもよい
- 表示フレット数の最小は 3 とする
- ただし実使用範囲が 4 以上であればそれに従う

### 表示例
- `x32010` → ローポジション表示
- `x68676` → `6fr` 開始、3 フレット表示
- `x68899` → `6fr` 開始、4 フレット表示

## 押弦編集機能

### 編集可能要素
各弦について、以下のいずれかを設定できる。

- ミュート
- 開放弦
- 任意のフレット押弦

### 編集ルール
- 1 弦につき 1 つだけ発音状態を持つ
- 別のフレットを選択したら、その弦の以前の状態は置き換える
- ミュートと開放弦も他の状態と排他的に扱う

### 編集結果の反映
押弦編集後、以下を再計算する。

- 実際に鳴る音
- ユニーク音
- ベース音
- コード名候補
- オンコード表記
- 表示フレット範囲

---

## 音名計算

### 基本
各弦の開放音とフレット番号から実音を算出する。

### 処理
- 開放弦の音高を基準に半音加算する
- 音名は 12 音クラスとして扱う
- コード判定ではオクターブ違いは同一音として扱う

### 出力
- 各弦の実音一覧
- ユニーク音一覧
- ベース音

---

## コード名判定

### 判定対象
現在鳴っている音からコード名候補を判定する。

### 基本手順
1. 鳴っている音を集める
2. ユニーク音に変換する
3. すべてのユニーク音をルート候補として試す
4. 各ルート候補について相対音程セットを計算する
5. 内部辞書のコード種別と照合する
6. 一致する候補を一覧化する
7. 優先順位に従って第一候補を決定する

### 優先順位
- 構成が単純なものを優先
- 一般的なコード名を優先
- ベース音との整合性を考慮する
- 不要に複雑な解釈は下位候補に回す

### 表示
- 第一候補を現在のコード名として表示する
- 第二候補以下は候補一覧に表示する

---

## オンコード判定

### 判定方法
- 実際に鳴っている最低音をベース音とする
- コード本体のルート音とベース音が異なる場合は `/bass` 表記にする

### 例
- 本体が C、最低音が E → `C/E`
- 本体が D、最低音が F# → `D/F#`

### 注意
- コード本体判定とベース音判定は分離して扱う
- 最低音を先にルートと決め打ちしない

---

## 候補コード名一覧
複数解釈が成立する場合は候補一覧を表示する。

### 例
- `C/E`
- `Em(#5)`
- `C`

初期実装では 3 件程度まで表示できればよい。

## 複数コード管理

### 概要
単一コードだけでなく、複数のコードダイアグラムを並べて管理できるようにする。

### 各コードブロックが持つ情報
- 一意な ID
- コード名
- 押弦データ
- ベース音
- 表示開始フレット
- 表示フレット数
- 横位置
- 並び順
- 間隔調整値

---

## レイアウト編集機能

### 基本
複数のコードダイアグラムを横方向に並べて表示する。

### できること
- コードブロックの追加
- コードブロックの削除
- コードブロックの複製
- コードブロックの並び替え
- コードブロックの左右位置調整
- コードダイアグラム同士の間隔調整

### 位置調整方法
初期実装では以下のいずれか、または両方を採用する。

- ドラッグによる移動
- 数値または簡易操作による左右オフセット調整

---

## 歌詞行との対応

### 基本
歌詞 1 行を表示し、その上にコードダイアグラムを配置できるようにする。

### 仕様
- 歌詞は単純な 1 行テキスト入力とする
- コードブロックは歌詞の上に配置する
- 各コードの左右位置を手動調整できる
- 譜割りの自動解析は行わない

### 目的
- 語頭や音節に合わせてコード位置を手で合わせられるようにする
- コード間隔と歌詞位置の両方を調整できるようにする

---

## 状態管理

### 最低限管理する状態
- 現在選択中のルート音
- 現在選択中のコード種別
- 現在選択中のフォーム候補
- 単一コードの押弦状態
- 現在のコード名候補一覧
- 複数コード配列
- 歌詞行
- レイアウト位置情報

### 保存
初期実装では永続保存は必須ではない。  
ただし設計上は将来の JSON 保存に拡張しやすい形にする。

---

## データ構造方針

### 押弦データ
各弦について以下のいずれかを持つ。

- `x`
- `0`
- 正の整数フレット番号

例:
- `["x", 3, 2, 0, 1, 0]`

### コードブロック
例:
- `id`
- `name`
- `frets`
- `bassNote`
- `startFret`
- `fretCount`
- `xOffset`

### コードタイプ辞書
コード種別名と相対音程配列を対応させる。

## 非機能要件
- ローカルで軽快に動作すること
- 状態変更後の再描画が即時反映されること
- UI は過度に複雑にしないこと
- 初期実装では機能を絞り、拡張しやすさを優先すること
